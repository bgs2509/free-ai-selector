# ADR Template

> Шаблон для Architecture Decision Records (ADR).

## Формат

Используем упрощённый формат ADR:

```markdown
# ADR-XXXX: Название решения

> Краткое описание (1 строка)

## Статус

[Proposed | Accepted | Deprecated | Superseded by ADR-YYYY]

## Контекст

Описание проблемы или ситуации, требующей решения.
- Какие требования?
- Какие ограничения?
- Какие существующие системы затрагиваются?

## Решение

Описание принятого решения.
- Что было выбрано?
- Как это работает?

## Альтернативы

### Альтернатива 1: Название

**Плюсы:**
- ...

**Минусы:**
- ...

### Альтернатива 2: Название

**Плюсы:**
- ...

**Минусы:**
- ...

## Последствия

### Положительные

- ...

### Отрицательные

- ...

### Риски

- ...

## Ссылки

- [Связанная документация](link)
- [Issue/PR](link)
```

---

## Правила нумерации

| Формат | Пример | Описание |
|--------|--------|----------|
| `ADR-XXXX` | ADR-0001 | 4 цифры с ведущими нулями |

---

## Жизненный цикл

```
Proposed → Accepted → [Deprecated | Superseded]
```

| Статус | Описание |
|--------|----------|
| Proposed | На рассмотрении команды |
| Accepted | Одобрено и применяется |
| Deprecated | Устарело, не рекомендуется |
| Superseded by ADR-XXXX | Заменено новым ADR |

---

## Когда создавать ADR

Создавайте ADR для:

- Выбора технологии или фреймворка
- Архитектурных паттернов
- Интеграций с внешними системами
- Значительных изменений в API
- Выбора алгоритмов/формул
- Изменений в структуре данных
